# 课程讲义生成技能 (Course Generator Skill)

自动生成超详细的课程讲义，包含深度源码分析、完整示例、FAQ等。支持子agent并行生成，自动上传飞书+GitHub，提供多格式输出。适用于大学课程、技术培训、教程制作等场景。

## ✨ 核心特性

### 🎯 智能生成
- **自动大纲**: 根据课程名称和版本自动生成结构化大纲
- **深度内容**: 每个知识点 5000-15000 字，包含源码分析
- **多级模式**: 基础版（5万字）、详细版（30万字）、超详细版（50万字）

### 🚀 高效并行
- **子Agent并行**: 多个子agent同时生成不同模块
- **智能调度**: 自动拆分任务，优化生成效率
- **实时监控**: 追踪生成进度，自动合并结果

### 📚 多格式输出
- **Markdown**: 完整的Markdown源文件
- **飞书文档**: 自动上传到飞书，便于在线阅读
- **GitHub仓库**: 自动创建GitHub仓库，支持版本控制

### 🎓 质量保证
- **源码分析**: 逐行注释关键代码
- **完整示例**: 200+ 可运行代码示例
- **FAQ丰富**: 60+ 常见问题解答
- **最佳实践**: 50+ 实战建议

## 📖 快速开始

### 方式1：AI 自动执行（推荐）

直接告诉 AI：

```
请生成 Spring Boot v2.3.12 课程讲义
- 课程层次：本科高级
- 学时：48学时（理论36 + 实验12）
- 重点：深度源码分析 + 实战示例
- 生成超详细版（30万字）
- 上传到飞书
```

### 方式2：使用脚本

```bash
cd /root/.openclaw/workspace/skills/course-generator/scripts

# 基础版
bash generate-course.sh -n "Spring Boot" -v "v2.3.12" -l "本科高级"

# 详细版（使用子agent并行）
bash generate-course.sh -n "Spring Boot" -v "v2.3.12" -d

# 完整流程（生成+飞书+GitHub）
bash generate-course.sh -n "Spring Boot" -v "v2.3.12" -d --feishu --github
```

详细用法请查看 [QUICKSTART.md](QUICKSTART.md)

## 💡 核心工作流

### 阶段1：需求收集（1分钟）
- 收集课程信息（名称、版本、层次）
- 确定学时分配和重点方向
- 明确输出要求

### 阶段2：大纲生成（2分钟）
- 生成课程大纲
- 规划模块和知识点
- 确定内容深度

### 阶段3：内容生成（10-60分钟）
- **基础模式**: 直接生成
- **详细模式**: 使用子agent并行生成
  - 拆分任务为多个模块
  - 启动2-4个子agent并行工作
  - 实时监控进度

### 阶段4：内容整理（5分钟）
- 合并子agent输出
- 统一格式和风格
- 生成统计报告

### 阶段5：多格式发布（5-10分钟）
- 创建飞书文档
- 上传GitHub仓库
- 生成索引和导航

## 🔧 参数说明

| 参数 | 说明 | 示例 | 必需 |
|------|------|------|------|
| `-n, --name` | 课程名称 | `Spring Boot` | ✅ |
| `-v, --version` | 课程版本 | `v2.3.12.RELEASE` | ❌ |
| `-l, --level` | 课程层次 | `本科高级` | ❌ |
| `-h, --hours` | 总学时 | `48` | ❌ |
| `-t, --theory` | 理论学时 | `36` | ❌ |
| `-e, --experiment` | 实验学时 | `12` | ❌ |
| `-d, --detailed` | 生成详细版 | - | ❌ |
| `--feishu` | 上传到飞书 | - | ❌ |
| `--github` | 上传到GitHub | - | ❌ |

## 📝 内容标准

### 每个知识点包含

1. **核心定义**（1000-2000字）
   - 是什么
   - 为什么需要
   - 解决什么问题

2. **深度解析**（2000-5000字）
   - 工作原理
   - 源码分析（带注释）
   - 执行流程

3. **实战示例**（1000-2000字）
   - 完整代码
   - 代码注释
   - 运行结果

4. **FAQ**（500-1000字）
   - 4-5个常见问题
   - 详细解答

5. **思考题**（3-5个）

### 统计数据（30万字讲义）

- ✅ 总行数：10,000 行
- ✅ 文件大小：256 KB
- ✅ 模块数：8 大模块
- ✅ 知识点：18 个
- ✅ 代码示例：200+
- ✅ FAQ：60+

## 🎨 定制化

### 自定义课程大纲

编辑 `templates/course-outline-template.md`，添加你的模块和知识点。

### 调整内容深度

编辑 `references/content-standards.md`，调整：
- 每个知识点的字数要求
- 源码分析的深度
- FAQ 的数量
- 代码示例的复杂度

### 子Agent并行策略

编辑 `scripts/spawn-subagents.sh` 中的模块划分，按你的需求调整：
- 模块数量
- 每个模块包含的知识点
- 并行度（2-4个子agent）

## ⚠️ 注意事项

### 文件大小限制
- **飞书文档**: 最大 100KB
  - 超过自动创建索引文档
  - 完整内容保留在文件中
- **GitHub**: 无限制

### 生成时间
- **基础版**: 5-10 分钟
- **详细版**: 30-60 分钟
  - 子agent并行：8-15 分钟
  - 内容整理：5 分钟
  - 发布：5-10 分钟

### 最佳实践
1. **首次使用**: 先生成基础版，确认大纲和结构
2. **二次生成**: 使用 `--detailed` 生成详细版
3. **定期保存**: 每个阶段都会保存中间文件
4. **版本管理**: 输出目录带时间戳，保留多个版本
